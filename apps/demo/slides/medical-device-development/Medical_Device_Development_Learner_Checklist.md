# Medical Device Development - Learner Checklist

Use this on one real problem from your setting, facility, or health system. The goal is not the most advanced device. The goal is a device or digital-health tool that can be used safely, maintained realistically, and adopted after the pilot ends.

## 1. Intended-Use Test

Write the exact claim before you design.

| Question | Your Answer |
|---|---|
| What patient or user problem are we trying to solve? | |
| Who is the intended user? Patient, nurse, doctor, OT, lab, pharmacy, EMS, technician, manager? | |
| What setting will it be used in? Ward, rural clinic, ambulance, lab, home, pharmacy, HMIS office? | |
| What medical decision or action could this influence? | |
| If the device or software is wrong, late, unavailable, or misunderstood, what harm could happen? | |

Rule: the object does not decide whether it is a medical device. The intended use, claim, user, setting, and possible harm decide.

## 2. Five-Family Checklist

| Family | Question to Ask | Evidence or Artifact |
|---|---|---|
| Need | Is this a real patient, staff, or system problem worth solving? | Observation notes, need statement, user quote |
| Translate | What is the simplest safe prototype that fits local power, staff, repair, and supplies? | Prototype sketch, requirements, test plan |
| Regulate | What claim, risk class, tests, technical file, and reliance route are needed? | Intended use, risk file, verification, validation |
| Value | Who pays, who saves time, who maintains it, and what gets replaced? | Cost story, procurement path, maintenance owner |
| Adopt | Will it still be used after the pilot ends? | Training plan, workflow map, ownership, data governance |

One-line takeaway: if one family is weak, the device may fail even if the technology works.

## 4. One Harm, One Mitigation

| Prompt | Your Answer |
|---|---|
| What is one way this device or software could injure a patient or delay care? | |
| Is the harm low, moderate, or high severity? | |
| What design control reduces the risk? | |
| What training, label, or workflow rule reduces the risk? | |
| What must be monitored after launch? | |

Rule: high-severity harm must be controlled before a prototype meets a patient.

## 5. Ownership Question

Complete this before accepting, donating, buying, or launching a device.

| Decision | Answer |
|---|---|
| Primary owner after the project ends | |
| Backup owner if the champion leaves | |
| Person or unit responsible for maintenance | |
| Budget owner for consumables, repair, and replacement | |
| First small test we can run in 30 days | |

Closing check: would this device still work if the donor leaves, the champion changes jobs, the internet fails, or a spare part breaks?
